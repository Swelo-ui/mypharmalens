# Comprehensive Security Plan: The "Ironclad" Free Tier Defense

This plan implements a multi-layered security system to prevent fake email signups and abuse of the 5-identification limit, using only free tools and Supabase's free tier capabilities.

## The 5 Distinct Identification Methods (Zero Cost)

We will layer these 5 methods to create a robust defense:

1. **Cloudflare Turnstile (Smart CAPTCHA)**: Stops automated bots from submitting the signup form. (Free unlimited)
2. **Device Fingerprinting (Browser-based)**: Generates a unique ID for the user's device using `@fingerprintjs/fingerprintjs` to detect multiple accounts from the same laptop/phone.
3. **IP Address Rate Limiting**: Tracks and limits the number of signups allowed from a single IP address within a 24-hour window using a custom tracking table.
4. **Honeypot "Trap" Fields**: Invisible form fields that legitimate users can't see but bots will fill out, instantly revealing them.
5. **Disposable Email Domain Blocklist**: A lightweight check against known "trash mail" providers (e.g., yopmail, tempmail) to force legitimate email usage.

## Technical Implementation Plan

### Phase 1: Database Infrastructure (Supabase)

We need a place to store signup metadata to enforce limits.

1. **Create** **`security_logs`** **table**:

   * Fields: `id`, `ip_address`, `device_fingerprint`, `email_domain`, `event_type` ('signup'), `created_at`.

   * **RLS Policy**: Insert-only for the Edge Function (Service Role); not readable by public.
2. **Create** **`security_rules`** **table (Optional config)**:

   * To store rate limits dynamically (e.g., "max\_signups\_per\_ip = 3").

### Phase 2: The "Gatekeeper" Edge Function (`secure-signup`)

We will move the signup logic from the Frontend to a secure Server-side Edge Function.

1. **Create** **`supabase/functions/secure-signup/index.ts`**:

   * **Inputs**: `email`, `password`, `turnstile_token`, `device_fingerprint`, `honeypot_value`.

   * **Step 1: Honeypot Check**: If `honeypot_value` is present -> Reject immediately (return success to confuse bot).

   * **Step 2: Turnstile Validation**: Validate the token with Cloudflare's API.

   * **Step 3: Disposable Email Check**: Regex match against top 100 disposable domains.

   * **Step 4: Rate Limit Check (IP & Device)**:

     * Query `security_logs`: "How many signups from this IP/Device in the last 24h?"

     * If > 3 (Configurable) -> Reject with "Too many accounts created".

   * **Step 5: Create User**: Call `supabase.auth.admin.createUser()` to actually sign them up.

   * **Step 6: Log Event**: Insert record into `security_logs`.

### Phase 3: Frontend Integration (`AuthForm.tsx`)

1. **Install Libraries**:

   * `@fingerprintjs/fingerprintjs` (Free, open source).

   * `react-turnstile` (or similar wrapper).
2. **Update UI**:

   * Add `<Turnstile />` widget.

   * Add hidden input field (Honeypot).
3. **Update Logic**:

   * Initialize FingerprintJS on mount.

   * Replace `supabase.auth.signUp()` call with `supabase.functions.invoke('secure-signup', ...)`

   * Handle new error messages (e.g., "Rate limit exceeded").

### Phase 4: Security & Cleanup

1. **RLS Policies**: Ensure no one can read the security logs.
2. **Cron Job (Optional/Manual)**: A SQL function to delete logs older than 30 days to save database space (Free tier limit: 500MB).

## Why This Works (Abuse Prevention)

* **"Infinite Fakesignup" Attack**: A user trying to create 10 accounts to get 50 identifications will be stopped after the 3rd account by **Device Fingerprinting** and **IP Limiting**.

* **Bot Attack**: Stopped by **Turnstile** and **Honeypot**.

* **Lazy Attack**: Stopped by **Disposable Email Blocklist** (forcing them to create real Gmails, which is slow).
